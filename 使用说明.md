# 瞌睡虫V5趋势分析软件 - 使用说明

## 一、软件概述

本软件是一款专业的彩票号码趋势分析工具，通过多维度数据分析和AI算法，帮助用户进行号码筛选和预测。主要功能包括：自动分割匹配、后台智能分割、AI智能分析、趋势图编辑等。

---

## 二、功能模块详解

### 2.1 开奖数据区域

**位置**：左侧面板顶部

**功能说明**：
- 显示最新一期开奖号码
- 显示数据获取时间
- 支持手动刷新开奖数据（每20秒自动刷新）

**数据格式**：
- 期号格式：`20260707-00037`（日期-期数）
- 开奖号格式：5位数字，系统取后3位进行分析

---

### 2.2 趋势图编辑器

**位置**：左侧面板中部

**功能说明**：
- **绘制趋势图**：在K线编辑器中手动绘制红色/绿色柱状图
- **保存趋势图**：保存自定义趋势图作为分割模板
- **导出/导入趋势图**：支持趋势图数据的备份和迁移

**趋势图规则**：
- **红色柱（中）**：号码组命中该期开奖号
- **绿色柱（未中）**：号码组未命中该期开奖号
- **模板含义**：末尾数字对应最新期，开头数字对应最旧期
- **示例模板**：`00000` 表示连续5期未中

---

### 2.3 自动分割匹配

**位置**：左侧面板，趋势图编辑器下方

#### 2.3.1 智能分割流程

**点击「开始智能分割」按钮启动**，弹出设置窗口：

| 设置项 | 选项 | 说明 |
|--------|------|------|
| 分割模式 | 自动分割 | 全自动执行所有分割步骤 |
| | 手动分割 | 第一步手动触发 |
| 匹配模式 | 模板模式 | 100%匹配趋势图模板 |
| | 趋势图模式 | 自定义相似度匹配 |
| 分割次数 | 1-4次 | 设置分割轮数，通常使用2次 |

**两步分割流程**：
1. **第一次分割**：根据选择的趋势图模板（如8中7），搜索符合条件的号码组
2. **第二次分割**：使用第一次分割的结果作为号码池，应用第二个模板（如4中3）

#### 2.3.2 开奖自动分割

**开关位置**：智能分割按钮下方

**功能逻辑**：
- 开启后，每次开奖数据更新时自动执行分割流程
- 首次需手动执行一次智能分割，系统保存配置作为模板
- 支持使用「⚡ 使用后台分割号码」作为第一次分割的号码来源

---

### 2.4 后台智能分割 ⚡

**位置**：自动分割匹配下方

**核心功能**：后台自动执行8中7分割，寻找5连挂最优号码组

#### 2.4.1 开启条件

点击开关开启，系统会立即执行一次分割循环（需至少6期开奖数据）

#### 2.4.2 工作流程（三步循环）

**第1步：搜索4连挂号码组**
- 使用模板 `0000`（连续4期未中）
- 使用**上上期**开奖号判断（如即将开50期，用48期判断）
- 目标：搜索并保存100组恰好4连挂的号码组

**第2步：过滤5连挂号码组**
- 使用**上期**开奖号检查（如即将开50期，用49期判断）
- 筛选出在上期仍未命中的号码组（即5连挂）
- **严格过滤**：只保留恰好5连挂的组，6连挂及以上全部淘汰

**第3步：评分选最优**
- 使用10维度评分机制对5连挂组进行评分
- 选取评分最高的一组作为最优组
- 显示在卡片中，提供复制功能

#### 2.4.3 持续循环

- 每次开奖数据更新后，自动重新执行完整循环
- 如果第一次分割选择了「⚡ 使用后台分割号码」，自动分割会等待后台分割完成后再执行

---

### 2.5 AI智能分析

**位置**：后台智能分割下方

#### 2.5.1 AI训练控制

| 按钮 | 功能 |
|------|------|
| 🎯 运行AI分析 | 对分割结果进行AI分析，生成推荐 |
| 🗑️ 清空结果 | 清空所有AI分析结果 |
| 🚀 开始训练 | 训练AI模式库（目标10000条） |
| 🔮 模式生成655注 | 使用模式库生成655注推荐号码 |
| 📋 清除统计 | 清除模式生成统计数据 |

#### 2.5.2 模式生成统计

**统计内容**：
- 生成次数：模式生成655注的总次数
- 命中次数：生成号码命中开奖号的次数
- 命中率：命中次数/生成次数
- 连中/连挂：连续命中/未命中的次数

**操作按钮**：
- 🗑️ 清除统计：清空所有统计数据
- 📋 复制号码：复制生成的655注号码

#### 2.5.3 评分阈值配置

**配置项**：
| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 第一轮阈值 | 30分 | 第一次分割结果评分低于此值时自动重试 |
| 第二轮阈值 | 30分 | 第二次分割结果评分低于此值时自动重试 |
| 最大重试次数 | 50次 | 达到此次数仍不满足条件则停止 |

**功能逻辑**：分割结果评分低于阈值时自动重试，直到满足条件

#### 2.5.4 模式库管理

**操作按钮**：
| 按钮 | 功能 |
|------|------|
| 📤 导出模式库 | 导出为JSON文件，方便跨设备迁移 |
| 📥 导入模式库 | 导入JSON文件，合并现有数据 |
| 🗑️ 清空模式库 | 删除所有模式数据 |
| 🧹 清空浏览器存储 | 清空所有本地存储数据 |

**训练机制**：
- 自动训练至10000条模式停止
- 空闲时每分钟进行模式验证
- 连续5次未中或命中率<15%的模式自动淘汰

#### 2.5.5 模式回溯验证

**配置项**：
| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 回溯期数 | 50期 | 验证模式在多少期历史数据中的表现 |
| 淘汰阈值 | 30% | 命中率低于此值的模式将被淘汰 |

**操作按钮**：
| 按钮 | 功能 |
|------|------|
| 🔍 执行回溯 | 手动执行一次模式回溯验证 |
| ⚡ 优化模式库 | 自动移除低质量模式 |
| 自动维护 | 每30分钟自动执行一次回溯验证 |

**验证逻辑**：
- 随机选取历史某一期验证模式
- 生成655注号码与下一期开奖号比对
- 命中则保留，连续未中则淘汰

#### 2.5.6 评分系统（10维度）

**核心指标（64%权重）**：

| 指标 | 权重 | 说明 |
|------|------|------|
| 回归期望值 | 24% | 基于连挂状态的回归概率，连挂越多回归期望值越高 |
| 趋势评分 | 20% | 趋势通道模型（布林带+EMA）分析结果 |
| 模式评分 | 20% | 与模式库中模式的匹配程度 |

**重要指标（10%权重）**：

| 指标 | 权重 | 说明 |
|------|------|------|
| 冷热平衡度 | 10% | 热号/温号/冷号的比例均衡度 |

**辅助指标（26%权重）**：

| 指标 | 权重 | 说明 |
|------|------|------|
| 最大连错 | 5% | 历史中最大连续未中的期数 |
| 近期表现 | 5% | 最近期数的表现评分 |
| 遗漏分数 | 5% | 热门遗漏号码的占比 |
| 多模型投票 | 5% | 多个模型一致推荐的占比 |
| 和值区间 | 3% | 号码和值是否在概率区间内 |
| 平均遗漏值 | 3% | 号码组平均遗漏期数，冷号组加分 |

#### 2.5.7 AI分析报告

**显示规则**：
- 总分低于20分的推荐不显示（过滤低质量推荐）
- 推荐卡片显示评分最高的号码组
- 卡片包含：综合评分、各维度评分、命中统计、复制按钮、查看趋势按钮

## 三、使用流程

### 3.1 基础使用流程

1. **导入数据**：确保系统已导入足够的历史开奖数据（至少830期）
2. **创建趋势图**：在K线编辑器中绘制分割模板
3. **手动分割**：点击「开始智能分割」执行首次分割
4. **开启自动分割**：首次分割完成后，开启「开奖自动分割」开关

### 3.2 后台智能分割使用流程

1. **开启后台分割**：点击「后台智能分割」开关
2. **等待搜索完成**：观察进度条，等待5连挂最优组生成
3. **设置第一次分割**：在分割配置中选择「⚡ 使用后台分割号码」
4. **开启自动分割**：开启后，每次开奖更新自动使用最新最优组

### 3.3 AI分析使用流程

1. **训练模式库**：点击「开始训练」，等待训练至10000条
2. **执行分割**：完成分割后自动触发AI分析
3. **查看推荐**：在AI分析报告中查看评分最高的推荐号码

## 四、数据持久化

### 4.1 自动保存

- **模式库**：自动保存到localStorage和后端（如有）
- **分割配置**：自动保存为模板
- **趋势图**：保存到localStorage

### 4.2 刷新恢复

- 刷新页面后，所有数据从localStorage自动恢复
- 模式库优先使用localStorage备份（防止后端数据丢失）

---

## 五、注意事项

1. **数据需求**：至少需要6期开奖数据才能使用后台智能分割
2. **4连挂概率**：每组8中7分割的4连挂概率约为1/4096，搜索可能需要较长时间
3. **评分阈值**：总分低于20分的推荐不会显示
4. **后台分割优先级**：当第一次分割选择「使用后台分割号码」时，自动分割会等待后台分割完成
5. **数据格式**：系统使用开奖号后3位进行分析，确保数据格式正确
